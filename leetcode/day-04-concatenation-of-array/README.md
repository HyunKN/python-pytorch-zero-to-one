# Day 04 — Concatenation of Array

## Problem

- LeetCode: [1929. Concatenation of Array](https://leetcode.com/problems/concatenation-of-array/)
- 정수 배열 `nums`가 주어졌을 때, `nums`를 두 번 이어 붙인 길이 `2n`인 배열 `ans`를 생성하여 반환합니다.

## 오늘 배울 내용

- Array
- Python 리스트의 동적 크기 및 연결 연산 (`+`)

## 구현 순서

1. 입력 배열 `nums`와 같은 크기의 빈 배열 혹은 미리 2배 크기로 채워진 배열을 생성합니다.
2. 각 원소를 순회하며 결과 배열의 `i`번째와 `i + len(nums)`번째 위치에 값을 할당합니다.
3. 파이썬다운 가장 간단한 방법으로 `nums + nums` 리스트 연결 연산을 사용하여 반환합니다.

검증은 LeetCode에서 수행하고 Accepted된 답을 [solution.py](solution.py)에 저장합니다.

## 직접 기록

### Approach

1. **이중 할당 방식 (enumerate)**:
   * `[0] * (len(nums) * 2)`로 2배 크기의 리스트를 먼저 생성하고, `enumerate(nums)`를 활용해 인덱스 `i`와 값 `num`을 받아서 `ans[i]`와 `ans[i + n]`에 각각 할당합니다.
2. **리스트 연결 연산 방식 (nums + nums)**:
   * 파이썬에서는 `+` 연산자로 리스트 두 개를 바로 이어붙일 수 있습니다. 내부적으로 C 레벨에서 최적화되어 동작하므로 속도가 매우 빠르고 직관적입니다.

### Complexity

- Time: `O(N)` — `nums` 배열의 길이만큼 원소를 복사합니다.
- Space: `O(N)` — 결과 배열 `ans`를 저장하기 위한 공간입니다. (결과 공간을 제외한 추가 메모리는 `O(1)`)

### Mistakes & 새로 알게 된 점 (Q&A 정리)

#### Q1. Python에서 `ans: List[int] = {}` 가능할까요?
* **불가능(타입 불일치)**합니다. `{}`는 빈 **딕셔너리(dict)**를 생성하는 기호입니다. (요소가 있으면 set이 될 수도 있음).
* `List[int]` 타입 힌트를 썼다면 빈 리스트인 `[]`로 초기화해야 합니다.

#### Q2. Python 리스트는 크기가 안 정해지고 넣을수록 계속 들어가나요?
* **네, 맞습니다.** 파이썬 리스트는 내부적으로 **동적 배열(Dynamic Array)** 구조입니다.
* 처음에 크기를 지정하지 않고 `append()` 등을 사용해 값을 계속 넣으면 자동으로 메모리 공간을 늘려주며 유연하게 확장됩니다.

#### Q3. 다른 언어들처럼 index `i`를 올려가며 for문을 도는 방법이 있나요?
* 파이썬에서는 `range()` 함수를 사용해 인덱스로 순회할 수 있습니다:
  ```python
  for i in range(len(nums)):
      # nums[i]로 접근 가능
  ```
* 인덱스 `i`와 원소 값 `num`이 둘 다 필요할 때는 **`enumerate()`**를 사용하는 것이 가장 파이썬답고 편리합니다:
  ```python
  for i, num in enumerate(nums):
      # i는 인덱스, num은 원소 값
  ```
  > **`enumerate()`란 무엇인가요?**
  > * `enumerate()`는 순회 가능한(iterable) 객체(리스트 등)를 인풋으로 받아, **(인덱스, 원소)** 형태의 튜플 쌍을 차례대로 반환해 주는 파이썬 내장 함수입니다.
  > * 인덱스 번호와 컬렉션의 실제 값을 동시에 다뤄야 할 때 루프 내부에서 별도의 카운터 변수를 늘려주는 번거로움을 줄여 줍니다.


#### Q4. 루프를 한 번만 돌아서 값을 넣어주는 방식이 다른 방식보다 더 효율적인가요?
* 시간 복잡도 관점에서는 둘 다 `O(N)`으로 동일합니다. (for문 2개 쓰기와 for문 1개쓰기 /2중말고)
* 하지만 파이썬에서는 `nums + nums`처럼 **내장 연산(C언어로 내부 구현된 고속 복사)을 사용하는 것이 속도와 가독성 면에서 훨씬 효율적**입니다.
