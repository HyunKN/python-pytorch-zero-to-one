# Day 06 — Two Sum

## Problem

- LeetCode: [1. Two Sum](https://leetcode.com/problems/two-sum/)
- 정수 배열 `nums`와 목표 정수 `target`이 주어졌을 때, 합해서 `target`이 되는 두 숫자의 인덱스를 반환합니다.

## 오늘 배울 내용

- **파이썬 딕셔너리(Dictionary) 구조와 작동 원리**:
  - **키-값(Key-Value) 쌍**: 사전(Dictionary)에서 단어를 찾아 뜻을 읽는 것처럼, 고유한 **Key**를 가지고 데이터인 **Value**에 바로 접근하는 구조입니다.
  - **해시 테이블(Hash Table)과 $O(1)$ 속도**: 
    - 리스트처럼 앞에서부터 하나씩 탐색(순차 검색)하지 않고, **해시 함수**를 통해 Key값에 대응하는 고유한 메모리 주소를 즉시 계산해 냅니다.
    - 덕분에 데이터의 개수가 아무리 많아져도 원하는 값을 찾을 때 단 한 번에 찾아가는 **$O(1)$(상수 시간)**의 성능을 자랑합니다.
    - 이번 알고리즘 최적화의 핵심으로, 이중 루프($O(N^2)$)를 단일 루프($O(N)$)로 줄이는 열쇠입니다.
  - **제약 조건**: 
    - Key는 고유해야 하므로 중복될 수 없습니다.
    - Key의 해시값 계산을 위해 값이 변하지 않는 **불변 객체(Immutable)**(문자열, 숫자, 튜플 등)만 키로 사용할 수 있습니다.
- **`enumerate()` 함수**:
  - 파이썬 내장 함수로, 리스트 등을 순회할 때 **인덱스(Index)와 값(Value)을 동시에 반환**해 줍니다.
  - 이 문제에서 딕셔너리에 숫자의 값과 그 위치(인덱스)를 짝지어 빠르게 저장하고 추적하는 데 사용됩니다.

## 구현 순서

1. `nums` 배열을 `enumerate()`로 순회하며 인덱스와 값을 가져옵니다.
2. 현재 값(`num`)에 대해 목표값(`target`)을 만들기 위해 필요한 짝꿍 숫자(`target - num`)가 딕셔너리(`seen`)에 있는지 확인합니다.
3. 딕셔너리에 존재하면 `[seen[짝꿍], 현재_인덱스]`를 즉시 반환합니다.
4. 존재하지 않는다면, 현재 값과 인덱스를 딕셔너리에 기록합니다 (`seen[num] = index`).

## 직접 기록

### Approach

- **브루트 포스 (기본 풀이)**: 이중 반복문을 활용해 모든 숫자 쌍의 합을 검사하는 방법으로, 시간 복잡도가 $O(N^2)$이 되어 배열이 클 때 매우 비효율적입니다.
- **해시 맵 (개선 풀이)**: 파이썬의 딕셔너리를 활용해 이미 확인한 숫자들을 기록해 둡니다. 키를 찾는 연산이 $O(1)$이므로 배열을 한 번만 순회($O(N)$)하면서 짝꿍 숫자를 바로 찾을 수 있습니다. 인덱스를 함께 얻기 위해 `enumerate()`를 활용했습니다.

### Complexity

- Time: $O(N)$ - 배열을 단 한 번만 순회하며, 딕셔너리 탐색 속도는 $O(1)$입니다.
- Space: $O(N)$ - 최악의 경우 배열의 모든 요소를 딕셔너리에 저장해야 합니다.

### Mistakes

- 처음에는 이중 반복문을 사용해 단순 비교하였으나 시간 복잡도가 $O(N^2)$으로 나타났습니다.
- 양수만 있을 것으로 오해하여 `nums[i] > target`일 때 `continue` 하려 했으나, 음수가 포함될 수 있는 조건에서는 이 방식이 오류를 유발할 수 있음을 배웠습니다. 해시 맵을 활용하면 이러한 예외 처리 없이도 가장 빠르고 명확하게 동작합니다.
