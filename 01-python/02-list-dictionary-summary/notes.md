## 접근 방식
- 상품 리스트(`products`)를 `for` 루프를 돌며 각 상품(딕셔너리)을 순회하며 요약 정보를 계산함.
- 전체 재고는 상품별 재고를 합산하고, 최고가는 현재 최고가보다 비싼 상품이 나올 때마다 최댓값과 상품명을 갱신하는 방식으로 추적함.
- 재고 부족 상품은 임계값 이하일 때 리스트에 추가함.

## 새로 배운 내용
- **`list[dict[str, object]]` 타입의 데이터 테이블 매핑**:
  - `list`의 대괄호 `[]`는 테이블 전체의 행(Row)들의 목록을 의미함.
  - `dict`는 하나의 행(Row)을 의미하며, `Key: Value` 쌍으로 이루어짐.
  - Key의 타입인 `str`은 테이블의 열 이름(Column Header)을 의미하며, Value의 타입인 `object`는 열 아래 들어갈 각 셀의 값들을 의미함.
  - 문자열(`str`)과 정수(`int`) 등 서로 다른 타입이 한 행 안에 공존할 수 있는 이유는 이들이 모두 파이썬 최상위 객체인 `object`에 속하기 때문임.
- **타입 힌트 `:`와 대입 `=`의 차이**:
  - `:`(콜론)은 이 변수가 어떤 데이터 타입을 갖는지 알려주는 타입 힌트(설명) 역할을 하며, 실제 메모리에 객체를 생성하지 않음.
  - `=`(등호)는 실제로 빈 리스트 `[]`나 값 등을 메모리에 할당(생성)할 때 사용함.
  - 따라서 변수를 정의하고 타입 힌트를 주려면 `low_stocks: list[str] = []`와 같이 `:`와 `=`를 나누어 써야 함.
- **`float("-inf")` (음의 무한대)**:
  - 최댓값을 구하는 알고리즘의 안전한 초기값 설정을 위해 사용함. 어떤 실수가 들어오든 첫 번째 비교에서 무조건 참이 되어 첫 번째 상품의 가격으로 최댓값이 자연스럽게 갱신되도록 보장함.
- **`str(product["name"])` 등의 형변환**:
  - `dict[str, object]`에서 꺼낸 값은 타입 검사기 입장에서 정체를 알 수 없는 `object`이므로, 문자열 전용 리스트나 변수에 담을 때 타입 경고가 발생할 수 있음. 이를 명시적으로 형변환(Casting)해 줌으로써 정적 타입 검사기를 만족시키고 런타임 에러를 방지함.
- **리턴값 결정 시점**:
  - 함수의 타입 힌트인 `-> dict[str, object]`는 규격의 약속일 뿐이며, 실제 반환되는 딕셔너리는 함수 마지막에 `return` 문이 실행될 때 동적으로 조립되어 생성됨.

## 실수와 해결
- **타입 선언 및 할당 문법 에러**:
  - `low_stock = list[str] = []`와 같이 타입 지정과 할당에 모두 `=`를 중복 사용함. 이로 인해 타입 정의 객체 자체에 대입하려고 시도하게 되어 `TypeError: 'type' object does not support item assignment` 에러가 발생함.
  - 해결: 타입은 `:`로 표시하고 값 할당은 `=`로 명확하게 분리한 `low_stocks: list[str] = []` 형태로 수정하여 해결함.
- **딕셔너리 키 이름 오타**:
  - `most_expensive`를 `most_expoensive`로 오타를 작성해 반환 키 불일치로 테스트 통과를 실패함.
  - 해결: 오타를 제거하고 변수명을 일치시켜 해결함.
