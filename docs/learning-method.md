# Top-down 학습 방법

이 저장소는 문법을 모두 외운 뒤 프로젝트를 시작하는 방식이 아닙니다. 먼저 완성된 작은 프로그램을 실행하고, 입력이 어떤 과정을 거쳐 출력으로 바뀌는지 확인한 다음 같은 핵심 기능을 직접 다시 구현합니다.

## 한 단계의 진행 순서

### 1. 완성된 예제 실행

각 폴더의 `demo.py`는 실행 가능한 완성 코드입니다.

```powershell
python demo.py
```

처음에는 모든 줄을 이해하려고 하지 말고 다음 세 가지를 먼저 확인합니다.

- 입력은 무엇인가?
- 어떤 함수가 어떤 순서로 호출되는가?
- 출력은 무엇인가?

### 2. README와 demo 코드 읽기

README의 코드 흐름을 따라 `demo.py`의 함수와 연결합니다. 모르는 문법은 그 자리에서 짧게 확인합니다.

### 3. 직접 구현

`exercise.py`는 demo와 같은 함수 이름과 입출력 계약을 가지지만 핵심 구현이 비어 있습니다. demo를 닫고 `exercise.py`를 완성합니다.

### 4. 자동 검증

같은 폴더의 `check.py`를 실행합니다.

```powershell
python check.py
```

- `PASS`: 최소 요구사항을 만족함
- `FAIL`: error 종류와 실패 위치를 확인한 뒤 수정

검증 통과는 학습 완료와 같지 않습니다. 왜 코드가 동작하는지 설명할 수 있어야 합니다.

### 5. 학습 기록

`notes.md`에 다음 내용을 기록합니다.

- 내 풀이 방식
- 새로 배운 문법과 개념
- 발생한 error와 원인
- 다음 날 다시 구현했는지 여부

## 막혔을 때 순서

1. error message를 끝까지 읽습니다.
2. 입력값과 중간 변수의 type, shape, 값을 출력합니다.
3. README의 입출력 계약과 비교합니다.
4. 20~30분 뒤에도 해결되지 않으면 hint를 찾습니다.
5. 정답을 봤다면 코드를 닫고 다시 작성합니다.

## 완료 기준

한 단계는 다음 조건을 모두 만족할 때 완료로 표시합니다.

- `check.py`가 PASS
- 핵심 함수의 입력과 출력을 설명 가능
- 중요한 중간 값 또는 Tensor shape를 설명 가능
- 다음 날 핵심 부분을 다시 구현
- `notes.md` 작성
