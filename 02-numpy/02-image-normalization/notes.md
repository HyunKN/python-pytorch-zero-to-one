# 이미지 정규화(Image Normalization) Q&A 노트

이 노트는 이미지 정규화 과정에 사용되는 NumPy 문법 및 수학적 전처리 개념을 질문과 답변(Q&A) 형식으로 정리한 파일입니다.

---

### Q1. `ndim`이 무엇인가요?
**A.** **Number of Dimensions (차원의 수)**의 약자로, 배열이 몇 개의 축(Axis)을 가지고 있는지 정수 값으로 알려주는 NumPy 속성입니다.

* **예시**:
  * `a = np.array([1, 2, 3])` $\rightarrow$ 1차원 벡터이므로 `ndim`은 **`1`**
  * `image`가 `(256, 256, 3)` 형태의 이미지 배열 $\rightarrow$ 축이 3개이므로 `ndim`은 **`3`**
* **코드**: 
  ```python
  if image.ndim != 3:
      raise ValueError("image shape는 (height, width, channels)여야 합니다.")
  ```
  이 조건문은 입력된 이미지 데이터가 흑백(2차원)이 아닌, **채널(RGB)이 포함된 3차원 컬러 이미지**가 맞는지 검사하는 안전장치입니다.

---

### Q2. 이미지가 3차원이라는 건 무슨 뜻인가요? 가로, 세로, RGB 채널을 뜻하나요?
**A.** **네, 정확합니다!**
컬러 이미지는 가로 픽셀들, 세로 픽셀들, 그리고 각 픽셀마다 빨강(R), 초록(G), 파랑(B) 빛의 세기 정보를 담고 있는 3가지 데이터 축으로 구성됩니다.
NumPy 배열 형태로는 일반적으로 **`(세로/Height, 가로/Width, 채널/Channel)`** 순서로 차원이 배치됩니다.

---

### Q3. `mean.shape`를 출력하면 무엇이 반환되나요?
**A.** **`(3,)`** 이라는 1차원 튜플이 반환됩니다.
* `mean`은 R, G, B 채널에 해당하는 평균값 3개가 들어 있는 1차원 배열(예: `[0.485, 0.456, 0.406]`)입니다. 
* 파이썬에서는 원소가 1개인 튜플을 표현할 때, 일반 괄호와 구분하기 위해 쉼표(`,`)를 붙여 `(3,)` 형태로 표현합니다.

---

### Q4. `image.shape[-1]`에서 `-1`로 3을 불러오는 이유와 원리는 무엇인가요?
**A.**
1. **원리 (음수 인덱싱)**: 파이썬에서는 배열이나 튜플의 요소를 뒤에서부터 셀 때 음수 인덱스를 사용하며, **`[-1]`은 가장 마지막 원소**를 뜻합니다. 이미지 크기가 `(256, 256, 3)`일 때, 맨 마지막에 위치한 값인 채널 수 `3`을 가리키게 됩니다.
2. **이유 (유연성)**: 인덱스를 `image.shape[2]`와 같이 양수 인덱스로 고정(하드코딩)하기보다 `[-1]`을 쓰는 것이 훨씬 안전합니다. 나중에 이미지 여러 장을 한 번에 처리하기 위해 배치(Batch) 차원이 추가되어 `(Batch, Height, Width, Channel)` 형태(4차원)로 배열 모양이 바뀌더라도, `[-1]`은 여전히 가장 마지막에 위치한 **채널 수**를 정확히 가리키기 때문입니다.

### Q5. `np.any`는 무엇이며, `np.any(std == 0)`는 어떤 역할을 하나요?
**A.**
* **`np.any`**: 배열의 조건 중 **"단 하나라도 참(True)이 있는가?"**를 판단하는 함수입니다. (논리합 `OR` 연산과 같습니다.)
* **역할**: 정규화 공식에서는 분모에 표준편차(`std`)가 들어갑니다. 수학적으로 **0으로 나누는 오류(Division by Zero)**는 불가능하므로, `std` 값 중 단 하나라도 `0`이 포함되어 있다면 참(`True`)을 반환하여 사전에 에러를 발생시키고 차단하는 역할을 합니다.
  ```python
  if np.any(std == 0):
      raise ValueError("std에는 0이 포함될 수 없습니다.")
  ```

### Q6. `scaled = image.astype(np.float32) / 255.0`은 각각 무엇을 의미하나요?
**A.**
1. **`astype(np.float32)`**: 원래 `0 ~ 255` 사이의 정수(`uint8`) 형태인 이미지 픽셀 값을 소수점 연산이 가능하도록 32비트 실수형(`float32`)으로 변환(형변환)하는 작업입니다. (딥러닝 연산의 표준 실수 포맷)
2. **`/ 255.0`**: 픽셀 값들을 최댓값인 255로 나누어 **`0.0 ~ 1.0` 사이의 비율**로 변환(스케일링)하는 핵심 동작입니다. 인공지능 모델은 입력 데이터가 너무 크지 않은 실수 범위(`0.0 ~ 1.0`)일 때 훨씬 더 안정적이고 빠르게 학습합니다.

---

### Q7. 정규화 공식 `(scaled - mean) / std`에서 왜 평균은 빼고 표준편차는 나누나요?
**A.**
1. **평균을 빼는 이유 (Zero-Centering)**: **데이터의 중심점(평균)을 `0`으로 맞춥니다.**
   * 데이터가 전부 양수(`0.0 ~ 1.0`) 영역에만 치우쳐 있으면 인공신경망 학습(역전파) 시 가중치의 갱신 방향이 한쪽으로 강하게 쏠려 지그재그로 헤매며 학습 속도가 극도로 느려집니다. 
   * 평균을 빼서 **어두운 픽셀은 음수(-), 밝은 픽셀은 양수(+)**로 중심을 잡아주면 학습 효율이 대폭 향상됩니다.
2. **표준편차로 나누는 이유 (Scaling)**: **데이터의 퍼진 정도(변화폭)를 `1`로 통일합니다.**
   * 어떤 색상 채널(예: R)은 밝기의 변화 폭이 아주 넓고, 어떤 채널(예: G)은 변화 폭이 아주 좁다면 인공지능은 변화가 넓은 R 채널 정보에만 지나치게 민감하게 반응할 수 있습니다. 
   * 이를 각 채널의 표준편차로 나누어 줌으로써, 모든 채널의 변화 폭을 동일하게 통일해 모든 채널을 편견 없이 공평하게 학습하도록 만듭니다.

### Q8. `mean = [0.485, 0.456, 0.406]`과 `std = [0.229, 0.224, 0.225]`는 무엇이며, 왜 하필 이 숫자를 쓰나요?
**A.**
* **의미**: 이 값들은 1,400만 장 이상의 거대한 이미지 데이터셋인 **ImageNet(이미지넷)**의 모든 이미지들로부터 미리 계산해 낸 **R, G, B 채널별 평균(`mean`)과 표준편차(`std`) 통계값**입니다.
* **왜 이 값을 써야 하나요?**:
  * 우리가 실무나 실습에서 사용하는 컴퓨터 비전의 수많은 인공지능 모델(ResNet, VGG, MobileNet 등)은 대부분 이 **ImageNet 데이터셋으로 미리 학습(Pre-train)**된 가중치를 가져와 사용(전이학습, Transfer Learning)합니다.
  * 이미 학습된 인공지능 모델은 이 특정한 `mean`과 `std`로 정규화된 이미지 데이터의 분포에 완벽하게 적응되어 있습니다.
  * 따라서, 우리가 새로운 이미지를 이 모델에 입력할 때도 **모델이 공부했던 것과 똑같은 분포로 값을 맞춰주기 위해** 동일한 `mean`과 `std`를 적용해 정규화해야 모델이 오작동하지 않고 올바른 예측을 수행할 수 있습니다.

---

### Q9. 특정 데이터나 전혀 다른 성격의 데이터셋에서는 평균과 표준편차를 직접 새로 구해서 써야 하나요?
**A.** **네, 상황에 따라 다릅니다.**

1. **모델을 처음부터 직접 학습시킬 때 (Scratch Training)**:
   * 의료용 X-ray 필름, 공장 불량 검출 이미지, 열화상 카메라 사진 등 일반적인 자연 사진(ImageNet)과 **색감이나 분포가 완전히 다른 데이터**를 사용해 모델을 처음부터 직접 학습시킬 때는, **해당 데이터셋 전체의 평균과 표준편차를 직접 계산하여 정규화**하는 것이 성능 면에서 훨씬 좋습니다.
2. **이미 학습된 모델(Pre-trained Model)을 가져와 미세 조정(Fine-tuning)할 때**:
   * 비록 내가 학습시키는 데이터셋이 완전히 개별적인 특성을 가졌더라도, ImageNet으로 이미 학습된 모델 가중치를 기반으로 학습을 시작한다면 **ImageNet의 평균(`mean`)과 표준편차(`std`)를 그대로 사용하는 것이 원칙**입니다.
   * 이미 만들어진 모델의 초반 필터(가장자리 선, 질감 등을 감지하는 기초 필터)들이 ImageNet 분포에 맞게 훈련되어 있으므로, 데이터의 분포(정규화 통계값)를 바꿔버리면 이 기초 필터들이 오작동할 수 있기 때문입니다.
